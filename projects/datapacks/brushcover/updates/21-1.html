Sorry for the long wait, lads. I couldn't get things done fast enough due to the real life situations I'm having to deal with. Also because the internal tools needed to make all this needs a rework in order to add some of the new stuff seen in this update.
There's also been quite a bit of changes made to cover all the side-cases introduced by the releases the pack is designed for, so for an update to a simple Brush project, we got a lot of changes to cover. Here goes!

Changes:

  • Is now designed for Minecraft 1.21 and 1.21.1!
  • You can now hide everything in Suspicious Blocks!
  • You can also make empty Suspicious Blocks by brushing Sand/Gravel with nothing else on!
  • Unbreaking enchantment now affects each durability point lost instead of being grouped together like before.
    • This is how Vanilla does it - though it is not as efficient when it's run.
    • This also means you'd lose less durability per brush with Unbreaking now, on average.
  • Improved some effects to feel more convincing. These affect:
    • Brush breaking when hiding an item within Sand/Gravel.
    • Brush sounds once an item's successfully hid in Sand.
  • Now supports Brushes with custom durability values!
  • Hiding items now works regardless of players' interaction ranges.
    • Note that longer interaction ranges are laggier if used, so be sure your devices are up for it first!
  • Improved MC-260225 workaround yet again to be closer to Vanilla's mechanics!
  • Brush sounds are now in the Player sound category.
    • It sounds obvious now, but the details are often overlooked by most.
  • Made a major update to the Admin Book system!
  • In-game Settings can now be saved as a Preset.
    • Saved Presets can then be used across worlds and shared online as well!
  • Added 2 new built-in presets to try out: Impatient and Reversible
  • Cover Durability is now Cover Damage to better describe what it does.
  • New setting: Cover % Damage - damages the Brush based on its max durability.
  • Brush damage settings now accept negative values.
    • It doesn't affect the project much, so why not?
  • Removed "Cover Limits" and "Mine to Move" settings.
    • Having these settings causes a lot more extra problems that just isn't worth the time fixing.
  • Admin Books now remember where you're at, and opens on the last page edited!
    • This saves a lot of editing time that would otherwise be spent on reaching that page from the beginning.
  • Each page now has up to 9 lines for settings instead of 8.
  • New "Presets" page that you can now use to save/load Presets!
  • Reset Settings button now only applies to settings from that page, and has been renamed accordingly.
  • The Next and Back buttons are now always usable, even if not rendered.
    • This lets you loop around to the start or end, reducing clicks needed to navigate to pages in half.
    • It takes a while getting used to the fact that you can do this because they don't really render properly, I can't really fix that without using a proper mod for it.
  • Fixed rapidly tapping the Brush button causing items to be lost once brushing is finished.
  • Fixed items popping out of Suspicious Blocks too early, leading to blocks without items to brush through.

Extras:

  • Added Admin Commands! To use these commands:
    • Simply type /function tdp., and the game should give you suggestions on what to use.
      Side note: Any suggestions with underscores followed by numbers are for internal use only and won't work if run on their own.
    • Only those with the "Allow Commands" permission can use Admin Commands.
  • A lot of filenames have been changed to properly organize everything.
    • IDs have been removed and is now replaced by code names.
    • The format is now tdp:code_name/data, to reserve the old names for Admin Commands.
  • After 6 years of making datapacks, we finally have icons now!
  • Improved the look of info text messages.
  • Text files have been merged into one single Interactive Help File.
    • I have to learn 3 new languages to get all this to work as well as it did.
    • Not the best at it yet, but it works well enough for now.
  • The master background has been updated for major releases as always.
  • Codebase now uses Scripter 4.

Notes:

  • Currently custom enchantments aren't supported, primarily because there isn't an efficient method to detect what a custom enchantment does yet.
    • Also because those things are Experimental for some reason. Ouch.
