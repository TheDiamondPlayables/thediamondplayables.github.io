Changes:

  • Is now designed for Minecraft 1.21 and 1.21.1!
  • Added 113 new recipes to the Stonecutter! Here are the highlights:
    • The Crafter can now be salvaged for parts!
    • Cut Copper can now make Copper Grates.
    • Chiseled Copper can be cut back into Cut Copper.
    • Copper Bulbs can be salvaged.
    • Copper Door, Copper Grate and Copper Trapdoor can be cut back into Ingots.
    • Tuff, Polished Tuff, Tuff Bricks, and Chiseled Tuff can be cut into eachother.
  • Experience Orbs and Vexes can no longer be cut by Stonecutters.
    • The reasoning for these 2 is kinda similar, but I'll let you figure it out.
  • Villagers now resist all damage dealt by Stonecutters.
    • They still act as if these actually hurt them though, because I find it funny.
  • New in-game Settings for Sawblade Damage:
    • Damage Dealt: Damage dealt per Stonecutter slice.
    • Villager Damage: Damage dealt per Stonecutter slice on Villagers.
    • Damage Cycle: The time between each Stonecutter slice.
    • Damage Knockback: If this damage knocks you away from the Stonecutter.
    • Death Message: If the Death Message is announced when someone dies to a Stonecutter.
  • New Presets:
    • Classic: Sawblade Damage, as it was designed.
    • Instacutter: Instantly kills most entities with a hit of the blade.
    • Bouncy Blades: All living things damaged by Stonecutter now bounce around like maniacs. Bouncy castle anyone?
  • Sawblade's Death Messages are now more traceable in the server logs.
    • Makes things easier to track what happened if you had to look through that thing.
  • Bugfixes:
    • Fixed a bug that caused Stonecutters to not deal damage in Peaceful.
    • Fixed a rare bug that caused Stonecutters to announce death messages before the victim actually dies.

Extras:

  • Added Admin Commands! To use these commands:
    • Simply type /function tdp., and the game should give you suggestions on what to use.
      Side note: Any suggestions with underscores followed by numbers are for internal use only and won't work if run on their own.
    • Only those with the "Allow Commands" permission can use Admin Commands.
  • A lot of filenames have been changed to properly organize everything.
    • IDs have been removed and is now replaced by code names.
    • The format is now tdp:code_name/data, to reserve the old names for Admin Commands.
  • After 6 years of making datapacks, we finally have icons now!
  • Improved the look of info text messages.
  • Text files have been merged into one single Interactive Help File.
    • I have to learn 3 new languages to get all this to work as well as it did.
    • Not the best at it yet, but it works well enough for now.
  • The master background has been updated for major releases as always.
  • Codebase now uses Scripter 4.

Notes:

  • I have considered cutting Copper Blocks into Doors/Trapdoors, but ultimately backed out on it.
    • If the Stonecutter has more recipes like this, you'd lose gameplay value in having the Crafting Table and its upgrade, the Crafter.
    • Also because it doesn't make much sense logically - Stonecutter can cut it to make it look like Doors/Trapdoors, but looks aren't everything.
  • I personally wouldn't consider Tuff to be something one can finely turn into Gravel, even though it would be consistent with the existing logic.
    • In Vanilla, Tuff isn't renewable, so converting it into something that isn't Tuff just means that eventually you'd have no Tuff left.
    • Even if Tuff is pretty common stuff, this is still a possibility I needed to consider, and for options that have a potentially damaging application, it's best to not be the one who made that happen, ya know?
  • An application of the 1.20.5's internal changes makes it possible to cut blocks into smaller versions of itself (displayed as heads), but that's not a part of Sawblade.
    • Sawblade aims to improve the existing items' relationships by adding recipes that makes sense based on the game's logic, and making new head items and adding that here doesn't make a lot of sense.
    • I'm still doing something like this though as a separate pack later on down the line. It'll be a pretty interesting challenge, for sure.
  • I tried to make it so that the Stonecutter only hits if you're actually standing on the Saw bit.
    • Problem with this is that all the methods I got doesn't perform that well when scaled up. The performance loss would always prove too much to make this mechanic worth implementing.
    • I'll be back to this some other time, but for now implementing it is more trouble than it's really worth.
