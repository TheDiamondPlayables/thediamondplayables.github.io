Villager Names has been reworked to better fit the game's worlds, and to keep things easy to wrap your head around since these features are purely detail stuff.

Changes:

  • Is now designed for Minecraft 1.21 and 1.21.1!
  • Names are now split into two categories: It's either a Full Name, or a Nickname.
  • Nicknames is a new category of short, interesting but weird Villager names.
    • They don't speak our language after all, so their Nicknames are weird.
    • There are 10291 unique nicknames available currently.
    • If there are any names that sounds offensive, it's because they were essentially made by merging random bits of words together, and I don't have the resources to do a full profanity scan of all the generated results just yet. Sorry!
  • Added more names and Previously known as Family Names.house names to the Full Name category to increase its diversity.
    • There are now 30 million combinations of full names, more than enough to last a lifetime.
    • Also, names only convey gender meaning if you think of it as such. Villagers are genderless, after all.
  • Names are no longer colored differently based on its category.
    • This is to keep things simple, since colored names don't really mean anything in game anyway.
  • Some villagers can now be found without a name.
    • They do have names, they just don't want you to know it.
    • You can use Name Tag commands on them if you're really curious though.
  • Popular Names and Mojang Names category have been removed.
    • I find that having ties to real world just takes away from the feeling of being in the game's world.
    • Also because, I don't have a reason to keep up to date with famous people, or new people in the Credits. I mean, it doesn't help me get better in doing this stuff, so why bother?
  • New Name Tag command: /nickname, which gives the Villager a new nickname.
  • /rename now no longer has a chance to give nicknames to Villagers.
    • You can use the new /nickname for that.
  • /nameless now works properly again in wiping out a Villager's name.
    • I didn't ship this workaround before because past me had it set in their head that having a different UUID messes things up a lot.
    • It does have some breaking implications, but it only affects a very small number of people, if the specific case that led up to it even exist.
    • Also, I'll never wait for Mojang to fix the bug that broke my things ever again.
  • Fixed a bug causing Villagers to take on others' house names if they were really close to them when /rename is applied.
  • New advancement: Name a Villager "/nickname" to show their nicknames!Also Known As
  • All previous advancements have had reworks to be more unique!
  • Removed the 2 reference trading achievements.
    • Mainly because of the whole "having ties to the real world" nature of those advancements.
  • Updated the internal Villagers' format to prepare for future updates.
    • If you're updating from an older version, the pack will do this automatically for you.
  • Completely overhauled the project's inner workings, using new techniques to improve pack performance!
  • Villager Names now pauses itself in servers when no players are active.
    • This improves performance on servers with a lot of forceloaded chunks, but no one's around.
    • You only really see these names when you're online anyway.
  • Improved performance whenever a Name Tag command was used.

Extras:

  • Added Admin Commands! To use these commands:
    • Simply type /function tdp., and the game should give you suggestions on what to use.
      Side note: Any suggestions with underscores followed by numbers are for internal use only and won't work if run on their own.
    • Only those with the "Allow Commands" permission can use Admin Commands.
  • A lot of filenames have been changed to properly organize everything.
    • IDs have been removed and is now replaced by code names.
    • The format is now tdp:code_name/data, to reserve the old names for Admin Commands.
  • After 6 years of making datapacks, we finally have icons now!
  • Improved the look of info text messages.
  • Text files have been merged into one single Interactive Help File.
    • I have to learn 3 new languages to get all this to work as well as it did.
    • Not the best at it yet, but it works well enough for now.
  • The master background has been updated for major releases as always.
  • Codebase now uses Scripter 4.
